# 即宿科技官网设计规格

## 目标

建设一个面向 C 端普通订房用户的公司官网，同时保留酒店合作与商务联系入口。官网服务于「今夜有房」小程序，帮助用户理解当天尾房撮合模式，并引导扫码体验。

## 受众

主要受众是当天或临近入住时寻找高性价比酒店房间的用户。次要受众是希望清理临期库存的酒店经营者，以及需要了解公司主体与合规信息的商务合作方。

## 信息结构

1. 首屏：即宿科技与今夜有房，主 CTA 为扫码体验小程序，副 CTA 为酒店合作。
2. 用户价值：位置预算筛选、当晚入住、房费到店付、规则透明。
3. 订房流程：填写需求、查看房间、支付服务费锁房、到店入住。
4. 规则说明：16:00 释放库存、15 分钟取消窗口、线上服务费与到店房费分离。
5. 酒店视角：清尾房、不公开破价、固定 SKU、订单核验。
6. 公司背书：西安即宿信息科技有限公司、即宿科技、域名与备案信息预留。
7. 底部转化：小程序二维码、酒店入驻留资、商务联系方式。

## 视觉方向

整体采用可信、克制、有运营能力的本地生活科技公司气质。页面应贴近 C 端产品官网，不做后台系统风、融资路演风或传统 OTA 列表页。

设计读法：B2C 本地生活产品官网，面向普通订房用户和酒店合作方，使用克制可信的现代服务品牌语言，采用静态 HTML/CSS 与轻量交互。

参数：`DESIGN_VARIANCE 6`、`MOTION_INTENSITY 4`、`VISUAL_DENSITY 5`。

## 技术约束

仓库当前无前端代码和构建系统，首版采用静态站点，便于直接部署到阿里云服务器或 OSS/Nginx。代码结构保持简单，后续可迁移到 Next.js 或 Vite。

## 验收标准

- 首屏清楚表达今夜有房的 C 端价值。
- 页面同时包含用户视角、酒店视角和公司信息。
- 至少包含三个转化入口：扫码体验、酒店合作、商务联系。
- 移动端优先可读，导航在桌面端单行显示。
- 页面无夸张虚构数据，无 AI 紫蓝渐变，无通用三卡模板堆叠。
- 可通过本地静态服务器预览，并通过基础内容测试。
